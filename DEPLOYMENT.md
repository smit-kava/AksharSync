# Deploying AksharSync with GoDaddy & Vercel

This guide outlines the process of connecting your GoDaddy domain to your Vercel deployment.

## Step 1: Add Domain in Vercel
1. Log in to your [Vercel Dashboard](https://vercel.com/dashboard).
2. Select your **AksharSync** project.
3. Go to **Settings** > **Domains**.
4. Type your domain name (e.g., `aksharsync.com`) and click **Add**.
5. Vercel will recommend adding both the naked domain (`aksharsync.com`) and the `www` subdomain (`www.aksharsync.com`). Select the recommended option.

## Step 2: Configure DNS in GoDaddy
Vercel will provide you with the specific values needed. Typically, you have two methods:

### Method A: Using DNS Records (Recommended)
1. Log in to your [GoDaddy Control Center](https://dcc.godaddy.com/control/portfolio).
2. Click on your domain to open **Domain Settings**.
3. Select **DNS** > **Manage DNS**.
4. Update/Add the following records:

| Type | Name | Value | TTL |
| :--- | :--- | :--- | :--- |
| **A** | `@` | `76.76.21.21` | Default (1 Hour) |
| **CNAME** | `www` | `cname.vercel-dns.com` | Default (1 Hour) |

> [!IMPORTANT]
> If there is an existing `A` record with Name `@`, edit it instead of adding a new one. If there is an existing `CNAME` record with Name `www`, edit it.

### Method B: Using Vercel Nameservers (Easier)
If you want Vercel to manage all your DNS records:
1. In GoDaddy DNS management, look for **Nameservers**.
2. Click **Change** and select **I'll use my own nameservers**.
3. Enter the nameservers provided by Vercel (e.g., `ns1.vercel-dns.com`, `ns2.vercel-dns.com`).

## Step 3: Verification
1. Go back to the **Vercel Domains** settings page.
2. Vercel will periodically check if the DNS records have been updated.
3. Once verified, the status will change to **Valid Configuration** (Blue checkmark).
4. SSL certificates will be generated automatically.

## Troubleshooting
- **Propagation Delay**: DNS changes can take anywhere from a few minutes to 48 hours to propagate globally.
- **CAA Records**: Ensure you don't have CAA records that prevent Vercel (Let's Encrypt) from issuing an SSL certificate.
- **Existing Records**: Double-check that there are no conflicting A or CNAME records.
